4.1 React.js 소개
Meta(Facebook)이 개발한 오픈소스 JavaScript 라이브러리
대규모 웹 서비스의 UI를 더 편하게 개발하기 위해 만들어진 기술

기술적 특징
1. 컴포넌트를 기반으로 UI 표현
-header.js / main.js / footer.js로 모듈화
-중복 코드 제거
 중복 코드 발생 시 페이지의 수가 늘어날 수록 코드를 유지보수하기 힘듦
-컨포넌트 단위로 모듈화하여 개발

2. 쉬운 화면 업데이트 구현
-업데이트: 사용자의 행동에 따라 웹 페이지가 스스로 모습을 바꿔 상호작용 하는 것
-선언형 프로그래밍: 과정은 생략하고 목적만 간결히 명시하는 방법
 <-> 명령형 프로그래밍: 목적을 이루기 위한 모든 일련의 과정을 설명하는 방식
-각각의 컴포넌트에는 state 변수 존재
-선언형 프로그래밍으로 업데이트를 위한 복잡한 동작을 직접 정의할 필요 없이 특정 변수의 값을 바꾸는 것만으로도 화면을 업데이트 시킬 수 있음

3. 화면 업데이트의 빠른 처리
-화면 업데이트를 쉽게 구현할 수 있으면서 동시에 빠르게도 처리
-선수 지식: 브라우저 동작 과정 (Critical Fendering Path)
 JavaScript가 DOM을 수정하면 업데이트 발생
 <reflow> Layout을 다시 실행
 <Repaint> Painting 다시 실행 둘 다 오래 걸리는 과정
-DOM은 딱 한 번만 수정해야 좋은 개선
-동시에 발생한 업데이트를 모아 한 번에 DOM을 수정할 수 있도록
-하지만 서비스의 규모가 커질수록 힘들어짐
-React는 이 과정을 자동으로 처리
 (Virtual DOM)
-업데이트가 모인 Virtual DOM 1회 수정 후 Actual DOM으로 반영
