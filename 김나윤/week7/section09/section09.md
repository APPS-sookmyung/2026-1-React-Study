# useReducer 훅 배우기

1. useReducer란?
- 컴포넌트 내부에 새로운 State를 생성하는 React Hook
- 모든 useState는 useReducer로 대체 가능
- useState와의 주요 차이점
    - 1) 상태관리 방식
        - useState: 간단한 상태관리를 위한 훅.기본적으로 상태 변수와 그 상태를 업데이트하기 위한 함수를 제공. 주로 한 가지 상태 값이 필요한 경우에 적합.
        * 한계점: useState를 사용해 App컴포넌트에서 state들을 만들면 그 해당 state들은 이 App컴포넌트 내부에서만 접근 가능. 그래서 state와 관련된 코드 또한 App컴포넌트 내부에서만 작성되어야 했음. -> 만약 이 state값이 현재보다 더 복잡해진다면 이 컨포넌트안에 훨씬 더 긴 코드를 작성해야함.그러나, 이 리액트 컴포넌트의 주된 역할은 UI를 렌더링하는 것임. -> 주객전도 상황.. 렌더링하는 코드보다 상태 관리 코드가 더 많아짐. -> 상태관리 코드들을 컴포넌트 외부에 별도 함수로서 분리해줄 필요!!
        - useReducer: 복잡한 상태관리를 위한 훅. 상태 전환 로직을 중앙에서 관리할 수 있도록 도와주며, 여러 개의 상태 값을 하나의 객체로 관리 가능. 주로 상태변화가 복잡해지는 경우에 유용힘.
