# 최적화

- 웹서비스의 성능을 개선하는 모든 행위를 일컫음.
- 아주 단순한 것부터 아주 어려운 방법 까지 매우 다양함.
- 일반 웹서비스 최적화방법으로는 이미지, 폰트, 코드 파일 등 정적 파일 로딩개선, 컴포넌트 내부의 불필요한 연산, 함수 재생성, 리랜더링 방지 .....여러 가지 기술이 있다. 이번에는 *"리액트의 연산낭비"*를 줄이는데 초점을 맞추어 살펴봄.


### useMemo
메모제이션 기법을 기반으로 불필요한 연산을 최적화하는 리액트 훅
- 리액트 앱에서 연산 최적화는 대부분 **메모이제이션(Memoization)기법**을 이용한다. 메모이제이션은 말 그대로 '메모하는 방법'이다. 즉, *특정 입력에 대한 결과를 계산해 메모리 어딘가에 저장했다가, 동일한 요청이 들어오면 저장한 결괏값을 제공해 빠르게 응답하는 기술.* -> 불필요한 연산을 줄여 효율적, 알고리즘에서는 동적 계획법이라고 함.

## React.memo 
- React.memo를 이용하면 메모이제이션 기법으로 컴포넌트가 불필요하게 리렌더되는 상황을 방지할 수 있다. 


## useCallback
useCallback은 컴포넌트가 리렌더될 때 내부에 작성된 함수를 다시 생성하지 않도록 메모이제이션하는 리액트 훅이다. 

## 최적화할 때 주의할 점
1. 기능구현이 먼저, 최적화는 항상 마지막에
2. 모든것을 최적화할 필요는 없다.
