# React Context
- 컴포넌트간의 데이터를 전달하는 또 다른 방법. 
- 기존의 Props가 가지고 있던 단점(Props Drilling)을 해결 가능. 
- 여기서 Props의 단점은 컴포넌트 트리에서 데이터를 전달할떄 언제나 부모에서 자식으로 단방향으로 전달하는 것을 말함. 즉, **자식의 자식, 트리에서 2단계이상 떨어져 있는 컴포넌트에 직접 데이터 전달하는게 불가능.** 
-> 마치 드릴로 땅 파는 것 같다고 해서 **Props Drilling문제**라고 함. 이는 컴포넌트 사이의 데이터 교환 구조를 파악하기 어려움.


### context란

- 문맥이라는 뜻. 글에서 문맥이란 글이 지닌 방향성. 
A와 B가 동일한 문맥 아래에 있다 = A와 B가 동일한 목적(기능)을 가지고 있다
- 리액트에서 Context는 같은 문맥아래에 있는 컴포넌트 그룹에 데이터를 공급하는 기능이라는 의미로 사용됨.
- 이를 이용하면 매 단계마다 일일이 Props를 전달하지 않고도 컴포넌트 트리 전역에 데이터 공급할 수 있어서 Props Drilling문제 해결 가능
