# 감정 일기장

## 구성 페이지 

1. index페이지 : 일기가 피드 형태로 나타남.
2. new페이지: 새로운 일기 추가 페이지
3. dairy 페이지: 작성한 일기를 상세히 조회.
4. home 페이지: 사용자가 앱에 접속하면 처음으로 만나는 페이지.

## 페이지 라우팅(Page Routing)

**경로에 따라 알맞은 페이지를 렌더링 하는 과정** 
ex) /new -> new페이지 렌더링

브라우저(Client)가 요청 /blog를 하면 서버에서 blog페이지 반환, 브라우저는 사용자에게 블로그 페이지 렌더링  = 페이지 라우팅

#### 페이지 라우팅이 이뤄지는 원리
 
전통적인 웹서비스: MPA(Multi Page Application)방식으로 서버가 다수의 웹페이지를 보유하고 있고,  브라우저가 서버에 요청을 보내게 되면 서버 사이드 렌더링 방식으로 미리 만들어진 페이지를 응답한다.


**React.js는 이 방식을 따르지 않는다. 쾌적한 페이지 이동 제공이 어렵기 때문**
--> 페이지 이동 시 전부 제거하고 새롭게 그려짐. 화면이 깜빡이면서 매끄럽지 않은 이동. 서버에 들어오는 페이지 이동 요청이 많다면 페이지 부하 발생. => MPA방식은 비효율적.

 **대신 SPA(Single Page Application)방식을 택함.**'

#### SPA방식
- 기본적인 틀을 가지고 있는 단일 페이지를 가지고 있는 방식. 
- 그래서 렌더링할 페이지를 index파일에 작성하는 게 아니라 자바스크립트에 컴포넌트 형식으로 작성하게됨.
- 페이지 요청이 오면 기본 틀 페이지를 먼저 전달 후 모든 js파일을 Bundling(묶다)해서 브라우저에 후속으로 전달함.
- 브라우저는 Bundle JS File(=React App)을 직접 실행해 컴포넌트 렌더링.
- => 페이지 전체를 렌더링 하는게 아니라 필요한 컴포넌트 요소만 교체해서 효율적임.

#### 동적경로(Dynamic Segments)
- 동적인 데이터를 포함하고 있는 경로
- URL Parameter: /뒤에 아이템의 id를 명시 -> 예시: ~/product/1 
	- 아이템의 id 등 변경되지 않는 값을 주소로 명시하기 위해 사용됨.
- Query String: ?뒤에 변수명과 값 명시 -> 예시: ~/search?q=검색어
	- 검색어 등 자주 변경되는 값을 주소로 명시하기 위해 사용된다.
